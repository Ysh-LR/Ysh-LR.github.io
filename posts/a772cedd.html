基础知识

SQL——结构化查询语言——非过程化但也不是面向对象

是一种对关系数据库进行访问的数据操作语言

数据定义语言（DDL）

数据库对象：数据库【database】、数据库表【table】、索引【index】、视图【view】、触发器【trigger】、存储过程【function】

创建——create

创建表时，同时创建其属性以及列完整性约束

⼀个数据库不允许有两个表同名。在⼀个表中，可以定义多个列，但不允许有两个属性列同名。

第一种定义方式——只能定义单列主键

create table table_1
 (
  attribute_1 char(13) primary key,
  attribute_2 char(12) not null,
  attribute_3 data unique,
  attribute_4 text null check(attribute_4 in('喵喵'，'呼噜呼噜')),
  attribute_5 varchar(2) not null default '逢考必过'
 )

第二种定义方式——可以定义复合主键和外键

create table_2
(
 attribute_6 varchar(2) not null,
 attribute_7 varchar(5) not null,
 attribute_1 char(13) not null,
 attribute_8 char(2) not null check(attribute_8 in ('男性','女性')),
 constraint table2_PK primary key(attribute_6,attribute_7),
 constraint attribute_1_FK foreign key(attribute_1)
    references table_1(attribute_1)
    on delete cascade
    on update cascade
)

创建索引：create index <索引名> on <表名><(列名)>

create index index_1 on table_1 (name);

创建视图：create view<视图名>[(列名1),(列名2),…] as<select查询>

不指定列名默认为所有列。

删除——drop

DROP DATABASE语句不能⽤在存储过程、触发器、事务处理程序中。

修改——alter

更改名字：alter <对象> <对象原名称> rename to <新名称>

对于数据表：修改已有的列，则操作后加column,修改还不存在的列（添加）不用加add

#添加
alter table table_1 add name varchar(20) not null;
#删除
alter table table_1 drop column attribute_3
#修改
alter table table_1 alter column attribute_2 type money

连接查询

在选课管理系统数据库中，我们希望得到各个学院的教师⼈数信息。该操作需要关联教师信息表（Teacher）和学院信息表 （College），查询学院名称、教师⼈数，输出按名称降序排列。这时 需要采⽤连接查询⽅法实现两表关联查询，其查询 SQL语句如下。

select College.CollegeName as 学院名称，count(Teacher.CollegeID) as 教师人数
from College,Teacher
where College.CollegeID = Teacher.TeacherID
group by College.CollegeName
order by College.CollegeName DESC;

注意：分组列名与排序列名要一致。

————————————–外连接👇

**LEFT JOIN **:左外连接,即使右表中没有匹配,也从左表返回所有的行【向左看齐，左表有多少行满足查询条件最终就有多少行】

RIGHT JOIN：右外连接，即使左表中没有匹配，也从右表返回所有的行【向右看齐】

FULL JOIN：全外连接，只要其中一个表中存在匹配，就返回行。

…

1.创建触发器

使⽤PostgreSQL创建触发器的基本步骤如下。

（1）检查数据库中将要创建的触发器所依附的表或视图是否存 在，如果不存在，必须⾸先创建该表或视图。

（2）创建触发器被触发时所要执⾏的触发器函数，该函数的类型 必须是tringer型，是业务处理的逻辑实现。但要注意，有些关系数据 库不需要独⽴定义触发器函数，⽽是在创建触发器时，将业务处理⽤ 触发器的过程体来实现。

（3）创建触发器时，⼀般需要指明触发器依附的表、被触发的时 间、是⾏级还是语句级触发器，以及触发器执⾏需要满⾜的条件。

在学⽣成绩数据库中，学⽣信息表（Student）存储 的每个学⽣的基本信息，学⽣成绩信息表（Stu_score）存储学⽣课程考试信息。为了跟踪学⽣成绩表的成绩改变情况，创建⼀个触发器， 当学⽣成绩发⽣改变时，系统⾃动记录学⽣变化前后的成绩数据到 Audit_score表中。

第 ⼀ 步 ： 检 查 系 统 中 是 否 存 在 Student（sid,sname.sex.classid） 、 Stu_score(sid.cid.score) 和 Audit_score(username,sid.cid.updatetime oldscore,newscore)表，如果不存在，可以使⽤如下语句创建。 

create table Student(
sid char(13) not null,
sname varchar(8) not null,
sex char(2) not null check(sex in ('男性','女性')),
classid char(13) not null,
constraint Student_PK primary key(sid)
);
create table Stu_score(
sid char(13) not null,
cid char(13) not null,
score varchar(3),
    constraint Stu_score primary key(sid,cid)

create table Audit_score(
	username varchar(8),
    sid char(13),
    cid char(13),

#注意，这里TG_OP中操作必须是大写
create or replace function score_audit()
	returns trigger as $score_audit$
	begin 
		if(TG_OP='DELETE')then
			insert into Audit_score select user,old.sid,old.cid,now(),OLD.score;
			return OLD;
		if(TG_OP='INSERT')then
			insert into Audit_score select user,new.sid,new.cid,now(),null,new.score;
			return NEW;
		if(TG_oP='UPDATE')then
			insert into Audir_score select user,old.sid,old.cid,now(),OLD.score,new.score
			where old.sid = new.sid and old.cid=new.cid;
			return NEW;
			END IF;
			RETURN NULL;
			END;

以下程序使⽤不带参数的游标，查询Student表的学号、学⽣姓名和性别。 

create or replace function cursorDemo()
returns boolean as $body$
declare
	unbound_refcursor refcursor;
	vsid varchar;
	vsname varchar;
	vsgender varchar;
begin
	open unbound_refcursor for
		'select sid,sname,gender from Student';
		loop
			fetch unbound_refcursor into vsid,vsname,vsgender;
			if found then
				raise notice '%-,%-,%-',vsid,vsname,vsgender;
			else
				exit;
			end if;
		end loop;
		close unbound_refcursor;
		raise notice '取数据循环结束...';
		return true;
	exception when others then
		raise exception 'error-----(%)',sqlerrm;
	end;
	$body$language plpgsql;

③ 使⽤Connection对象的下列⽅法之⼀创建查询语句对象。

• Connection.createStatement()创建⼀个Statement对象， 实现静态SQL语句查询；

• Connection.prepareStatement(String sql) 创 建 ⼀ 个 PreparedStatement 对象，实现动态SQL语句查询；

• Connection.prepareCall(String sql) 创 建 ⼀ 个 CallableStatement对象，调⽤数据库存储过程。

（2）执⾏查询返回查询结果，如果不需向 SQL 语句传递动态参 数，则使⽤静态查询Statement对象。Statement每次执⾏SQL语 句，相关数据库都要执⾏SQL语句的编译，执⾏查询有如下3种形式。

① Statement.execute(String sql) 执⾏各种SQL语句，返回⼀ 个boolean值，true表⽰执⾏的SQL语句具备查询结果，可通过 Statement.getResultSet()⽅法获取；

② Statement.executeUpdate(String sql) 执 ⾏ SQL 中 的 INSERT、UPDATE、DELETE语句，返回⼀个int值，表⽰受影响的记录 数⽬；

③ Statement.executeQuery(String sql) 执 ⾏ SQL 中 的 SELECT语句，返回⼀个表⽰查询结果的ResultSet对象。

嵌入式SQL通信

1.向Java语⾔返回结果

数据库查询⼀般需要返回多条记录，则ResultSet接⼝对象⽤于返回查询结果集，该结果集本质上是内存中⽤于存储多条记录的游标， 主要有以下⼏种⽅法访问游标的记录信息。

① ResultSet.next()将游标由当前位置移动到下⼀⾏；

② ResultSet.getString(String columnName) 获取指定字段 的String类型值；

③ ResultSet.getString(int columnIndex) 获 取 指 定 索 引 的 String类型值；

④ ResuleSet.previous()将游标由当前位置移动到上⼀⾏。

2.向SQL语句传递参数 如果Java宿主语⾔需要向SQL语句传递参数，则使⽤动态查询 prepareStatement对象， preparedStatement预编译SQL语句， ⽀持批处理，执⾏查询有类似Statement对象的3种执⾏⽅式，且执 ⾏ ⽅ 法 中 没 有 参 数 ， 例 如：“prepareStatement.executeUpdate();”还可以批处理执⾏， prepareStatement对象使⽤addBatch()向批处理中加⼊⼀个更新语 句，使⽤executeBatch()⽅法成批地执⾏SQL语句，但不能执⾏返回 值是ResultSet结果集的SQL语句，⽽只是执⾏executeBatch()。

以下程序是使⽤批处理⽅法向表Stu_score插⼊学⽣ 的课程成绩，并使⽤动态查询语句查询成绩⼤于等于80分的学⽣课程成绩。

package testConnDB;
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

public class SQLinJava{
	public static void main(String[] args){
		Connection conn = null;
		string URL = "jdbc:postgresql://localhost:5432/testDB";
		String userName = "user";
		String pwd = "sa";
		String[] sid = {"0001","0002","0003","0004","0005"};
		string[] cid = {"1000","2000","3000","1000","4000"};
		int[] score = {90,80,98,100,62};
		try{
			Class.forName("org.postgresql.Driver");
			conn = DriverManager.getConnection(URL,username.pwd);
			System.out.println("连接成功！")；
			String insertsql = "insert into Stu_score(sid,cid,score) values(?,?,?)";
			String querySql="select sid,cid,score from Stu_score where score>=?";
			Preparedstatement psinsert = conn.prepareStatement(insertSql);
			Preparedstatement psQuery = conn.prepareStatement(querySql);
			for(int i = 0;i<sid.length;i++){
			psInsert.setString(1,sid[i]);
			psInsert.setString(2,cid[1]);
			psInsert.setInt(3,score[i]);
			psInsert.addBatch();
			}
			psInsert.executeBatch();
			psQuery.setInt(1.80);
			ResultSet rs =
			psQuery.executeQuery();
			while (rs.next()) { // 判断是否
				System.out.println(rs.getString("sid")+" "+rs.getString("cid")+ " " +rs.getInt("score"));
			}
			psQuery.close();
			psInsert.close();
			conn.close();
		} catch ( Exception e ) {
			System.err.println(
			e.getClass().getName()+": "+e.getMessage() );
			System.exit(0);
		}
	}
}

create or replace function sale_insert()
returns trigger
as$$
declare 
	book_remain int2;	
begin
	if(TG_OP='insert')then
        select stock.book_stock into book_remain from stock
        where new.isbn = stock.isbn;
		if(book_remain > new.sale_number)then
			update stock set book_remain = book_remain- new.sale_number
			where new.isbn=stock.isbn;
		else
			notify "库存不足";
			exit;
		end if;
		return new;
	end if;
    return null;
end;
